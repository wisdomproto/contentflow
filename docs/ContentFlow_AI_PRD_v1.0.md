## 2. 사용자 흐름 (User Flow)

아이디어를 입력하면 탭을 이동할수록 컨텐츠가 구체화되는 선형 워크플로우입니다.

```
[STEP 1] 기본 설정 탭          [STEP 2] 블로그 탭              [STEP 3] 카드뉴스 탭        [STEP 4] 영상 탭
폴더 선택 + 주제·키워드·   →   초안 → 네이버 최적화 글     →   블로그 → 핵심 슬라이드   →   블로그 → 릴스·쇼츠
톤 입력 → AI 통합 초안 생성     C-Rank / D.I.A+ / GEO 적용      5–10장 자동 추출             대본 변환
```

## 3. 기능 요구사항 (Functional Requirements)

### 3.0 폴더(대분류) 구조 및 페르소나 설정

컨텐츠는 **폴더(대분류) > 개별 컨텐츠** 2단계 계층으로 관리됩니다.  
C-Rank 핵심 전략인 주제 일관성을 폴더 단위로 강제하고, 폴더별로 고유한 페르소나와 블로그 설정을 부여합니다.

#### 3.0-A 폴더 개념 및 역할

| 항목 | 설명 | 예시 |
|------|------|------|
| **폴더 (대분류)** | 하나의 블로그 주제 영역 = 네이버 블로그 카테고리 1개에 대응 | 강남 맛집 / 육아 꿀팁 / 제주 여행 |
| **컨텐츠 (하위)** | 폴더 내에 생성되는 개별 포스팅·카드뉴스·숏폼 | 강남역 카페 투어 / 홍대 파스타 맛집 TOP5 |
| **폴더 ↔ C-Rank** | 같은 폴더 내 컨텐츠는 동일 주제로 묶여 C-Rank 전문성 점수에 누적 기여 | 맛집 폴더 글 10편 = 맛집 블로그 신뢰도 상승 |
| **폴더 ↔ 페르소나** | 폴더마다 독립적인 글쓰기 페르소나·톤앤매너·키워드 전략 보유 | 맛집 폴더: 친근한 / 육아 폴더: 따뜻하고 전문적 |

#### 3.0-B 폴더 생성 및 기본 설정

폴더를 처음 만들 때 아래 항목을 설정합니다. 이 설정값은 해당 폴더 내 모든 컨텐츠 생성 시 자동으로 적용됩니다.

| 설정 항목 | 설명 | 비고 |
|-----------|------|------|
| **폴더명** | 대분류 이름 (예: 강남 맛집, 육아 꿀팁) | 필수 |
| **폴더 색상 & 아이콘** | 사이드바 식별용 색상 및 이모지 아이콘 선택 | 선택, 기본값 자동 배정 |
| **블로그 카테고리 연결** | 네이버 블로그 카테고리명 매핑 (C-Rank 맥락 관리) | 권장 |
| **메인 키워드** | 이 폴더의 핵심 타겟 키워드 1–3개 (모든 컨텐츠에 공통 적용) | 권장 |
| **연재 시리즈 설정** | 시리즈 제목 및 순번 자동 부여 여부 (예: 강남 맛집 탐방 #1, #2…) | 선택 |

#### 3.0-C 폴더별 페르소나 설정

각 폴더는 독립적인 **글쓰기 페르소나**를 가집니다.  
페르소나는 Gemini AI 프롬프트에 시스템 역할로 주입되어 폴더 내 모든 컨텐츠 생성에 일관되게 적용됩니다.

| 설정 항목 | 옵션 / 입력 형식 | AI 적용 방식 |
|-----------|-----------------|--------------|
| **톤앤매너** | 친근한 이웃 / 전문적 정보형 / 유머러스 MZ / 따뜻한 육아맘 / 직접 입력 | 시스템 프롬프트 역할로 주입: '당신은 [톤] 스타일로 글을 쓰는 블로거입니다' |
| **1인칭 설정** | 나 / 저 / 필자 중 선택 | 본문 전체 1인칭 대명사 일괄 통일 |
| **블로거 소개** | 작성자 배경 한 줄 (예: '10년차 강남 직장인') | 도입부·마무리에 자연스럽게 삽입 |
| **서술 스타일** | 경어체 / 반말체 / 혼용 선택 | 문체 일관성 유지 프롬프트 적용 |
| **이모지 사용** | 많이 / 적당히 / 사용 안 함 중 선택 | 소제목·문장 끝 이모지 삽입 빈도 조절 |
| **금지어 설정** | 특정 단어·표현 블랙리스트 직접 입력 | 생성 후 후처리 필터로 제거 |

- **페르소나 미리보기**: 설정 저장 전 '샘플 문단 생성' 버튼으로 실제 문체 확인 가능
- **페르소나 공유**: 폴더 설정을 JSON으로 내보내 다른 계정·팀과 공유 가능
- **폴더 설정 편집**: 사이드바 폴더명 우측 ⚙ 버튼 클릭 → 설정 드로어(Drawer) 슬라이드 인

#### 3.0-D 사이드바 구조

Left Pane은 폴더 기반으로 재구성되며, 폴더와 하위 컨텐츠를 트리 구조로 표시합니다.

| 영역 | 구성 요소 | 동작 |
|------|-----------|------|
| **최상단** | [+ 새 폴더] 버튼 + 전체 검색창 | 폴더명·컨텐츠 제목 통합 검색, 검색 결과 폴더 경계 유지하며 표시 |
| **폴더 헤더** | 📁 [색상 아이콘] 폴더명  [컨텐츠 수]  [⚙] [▼/▶] | 클릭 시 하위 컨텐츠 목록 토글. ⚙ 클릭 시 폴더 설정 드로어 오픈 |
| **컨텐츠 아이템** | └ 📄 컨텐츠 제목  [채널 아이콘]  [상태 뱃지]  날짜 | 클릭 시 해당 컨텐츠 에디터로 이동. 우클릭 시 복제·이동·삭제 컨텍스트 메뉴 |
| **채널 아이콘** | 📝 블로그 / 🎴 카드뉴스 / 🎬 숏폼 — 생성 완료된 채널만 표시 | 아이콘 클릭 시 해당 탭으로 바로 이동 |
| **상태 뱃지** | 초안(회색) / 생성완료(파랑) / 발행(초록) / 수정필요(주황) | 필터 버튼으로 상태별 컨텐츠 목록 필터링 가능 |
| **미분류** | 사이드바 최하단 '미분류' 영역에 자동 배치 | 드래그앤드롭으로 폴더로 이동 가능 |

### 3.1 [탭 1] 기본 설정 (Idea & Source)

모든 컨텐츠의 원천 데이터가 생성되는 탭입니다.  
먼저 폴더를 선택하면 해당 폴더의 페르소나·키워드 전략이 자동 로드되어 입력 필드에 반영됩니다.

| 항목 | 설명 | 비고 |
|------|------|------|
| **폴더 선택** | 소속 폴더(대분류) 선택 — 선택 즉시 페르소나·메인 키워드 자동 로드 | 필수 |
| **주제 입력** | 자유 형식의 컨텐츠 주제 (예: 강남역 카페 투어) | 필수 |
| **핵심 키워드** | 직접 입력 또는 AI 자동 추천 (네이버 키워드 검색량 기반) | 선택 |
| **경험 데이터** | 사용자 직접 경험 메모 (주차·가격·분위기 등) | D.I.A+ 핵심 |
| **톤앤매너** | 폴더 페르소나에서 자동 로드, 이 컨텐츠에서만 개별 변경 가능 | 자동 |
| **참고 자료** | URL 또는 이미지 업로드 | 선택 |

- **AI 초안 생성 버튼**: 클릭 시 통합 ContentContext(JSON) 생성 후 블로그 탭으로 자동 이동
- **우측 패널 — 네이버 키워드 도구 API**: 입력 주제 기반 키워드별 월간 검색량·PC/모바일 비율·경쟁 블로그 수 실시간 조회
- **연관 키워드 클러스터링**: 검색량 상위 키워드를 메인·서브·롱테일로 자동 분류하여 추천
- **경쟁 블로그 분석**: 상위 노출 블로그 제목·구조·키워드 패턴을 Gemini가 분석하여 차별화 전략 제안

#### 3.1-A 네이버 API 연동 — 키워드 인텔리전스

단순 AI 추천을 넘어 실제 검색 데이터를 기반으로 키워드 전략을 수립합니다.

| API | 활용 목적 | 추출 데이터 |
|-----|-----------|-------------|
| **네이버 검색광고 API** (키워드 도구) | 키워드 검색량·경쟁도 조회 | 월간 검색량(PC/모바일), 경쟁 블로그 수, 클릭률 |
| **네이버 데이터랩 API** | 트렌드 및 시기별 관심도 분석 | 최근 90일 검색 트렌드, 연령·성별 분포 |
| **네이버 검색 API** (블로그) | 상위 노출 경쟁 블로그 수집 | 제목 패턴, 본문 구조, 사용 키워드 빈도 |

- **키워드 추천 로직**: 네이버 검색량 데이터 → Gemini 분석 → 메인·서브·롱테일 키워드 3단계 분류
- **경쟁도 시각화**: 키워드별 검색량 대비 블로그 포화도를 그래프로 표시, 블루오션 키워드 하이라이트
- **경쟁 블로그 분석**: 상위 5개 블로그 제목·소제목 구조를 Gemini가 파악하여 차별화 포인트 자동 제안
- **트렌드 알림**: 데이터랩 기준 최근 급상승 연관 키워드 실시간 뱃지 표시

### 3.2 [탭 2] 네이버 블로그 생성

탭 1 데이터를 네이버 블로그 포맷으로 렌더링합니다.  
본문은 **섹션** 단위로 구성되며, 네이버 **C-Rank / D.I.A+ / GEO** 세 가지 알고리즘을 모두 고려한 AI 생성 로직을 적용합니다.

#### 3.2-A AI 생성 로직 — C-Rank 대응 (블로그 신뢰도)

C-Rank는 블로그 전체의 주제 일관성과 전문성을 평가합니다.  
단일 포스팅이 아닌 블로그 전체 맥락을 고려한 콘텐츠 기획이 필요합니다.

| 항목 | AI 생성 로직 | 구현 방식 |
|------|-------------|-----------|
| **주제 일관성** | 메인 카테고리(예: 강남 맛집) 설정 후 세부 주제를 연재 형식으로 연결 | 폴더 메인 카테고리 참조, 이전 포스팅 주제 히스토리 기반 맥락 연결 |
| **전문성 강조** | 공신력 있는 수치·출처를 인용하는 문장 구조 자동 생성 | 정보성 글 생성 시 출처 명시 문장 포함 프롬프트 강제 적용 (예: '농림부 2024년 통계에 따르면') |
| **연재 기획 지원** | 관련 주제 시리즈 자동 제안 (예: 1편 방문 후기 → 2편 메뉴 심층 분석 → 3편 주변 가볼 곳) | 사이드바 '다음 편 아이디어 추천' 버튼으로 연재 시리즈 자동 기획 |

#### 3.2-B AI 생성 로직 — D.I.A+ 대응 (문서 독창성·사용자 경험)

D.I.A+는 개별 포스팅의 독창성과 사용자 체류시간을 딥러닝으로 평가합니다.  
작성자의 생생한 경험이 담긴 1인칭 문체와 흡입력 있는 구조가 핵심입니다.

| 항목 | AI 생성 로직 | 구현 방식 |
|------|-------------|-----------|
| **1인칭 경험 서술** | 기본 설정 탭의 '경험 데이터' 입력값을 본문에 자연스럽게 녹여 독창성 확보 | 프롬프트: '제가 직접 방문해 보니 [경험 데이터]가 인상 깊었습니다' 패턴으로 변환. 백과사전식 서술 금지 |
| **후킹 도입부** | 서론 첫 2–3문장에 독자 호기심을 강하게 자극하는 질문형·공감형 후킹 문구 자동 배치 | 후보 3개 제시 후 사용자 선택. 예: '강남역에서 웨이팅 없이 밥 먹을 수 있다고? 저도 처음엔 믿지 않았어요' |
| **리듬감 있는 문단** | 한 문단 최대 3–4문장으로 강제 분절, 섹션마다 소제목+이미지로 시각적 환기 제공 | 프롬프트: '한 문장은 모바일 기준 2줄 이내 단문, 문단은 4문장 이내로 제한' 규칙 적용 |
| **체류시간 유도 구조** | 본문 중간에 '더 보기' 유도 문구, 글 말미에 댓글·공유 CTA 자동 삽입 | 마지막 섹션에 '궁금한 점은 댓글로!' / '이웃 추가하면 다음 편 알림!' 패턴 자동 적용 |
| **LSI 키워드 혼합** | 메인 키워드 5–7회 자연 배치 + 유사 검색어·연관 단어 자동 삽입으로 스팸 방지 | 네이버 키워드 API 데이터 기반 연관어 목록 생성 후 Gemini가 문맥에 맞게 분산 배치 |

#### 3.2-C AI 생성 로직 — GEO 대응 (AI 검색 노출, 2026 최신)

네이버 AI 검색(큐:)은 질문에 대한 정답이 명확한 문서를 우선 인용합니다.  
전통 SEO를 넘어 AI 검색 결과에 출처로 노출되는 GEO 전략을 반영합니다.

| 항목 | AI 생성 로직 | 구현 방식 |
|------|-------------|-----------|
| **Q&A 블록 자동 삽입** | 본문 내 주요 질문을 추출해 FAQ 형식으로 재구성한 Q&A 블록 자동 생성 | 예: 'Q. 강남역 주차 가능한가요? A. 네, 건물 지하 2층 주차장 이용 가능합니다 (2시간 무료).' 섹션 중간 삽입 |
| **핵심 요약 블록** | 글 마지막에 '오늘의 핵심 요약' 3줄 요약 블록 자동 생성 | AI 검색 로봇이 답변 출처로 인용하기 좋은 구조화된 요약. 인용구 HTML 태그로 강조 처리 |
| **명확한 정보 구조** | 숫자·고유명사·날짜 등 구체적 팩트를 본문에 우선 배치 | 프롬프트: '막연한 추측 표현(~인 것 같다, ~라고 하더라) 금지, 구체적 수치 우선 사용' 규칙 강제 |
| **제목 최적화** | 핵심 메인 키워드를 제목 맨 앞에 배치, 숫자·감성 수식어 조합으로 클릭률 극대화 | 제목 후보 3개 자동 생성. 예: '강남역 맛집 TOP5 \| 웨이팅 없이 먹은 솔직 후기' |

#### 3.2-D 섹션(Section) 구조 상세

블로그 본문은 **섹션** 단위로 구성됩니다.  
각 섹션은 **이미지 1장 + 텍스트 블록**의 조합이며, 섹션 단위로 편집·이동·삭제가 가능합니다.

| 구성 요소 | 설명 | 비고 |
|-----------|------|------|
| **섹션 헤더** | 소제목 텍스트 (볼드·이모지 자동 적용) | 선택 |
| **이미지 블록** | 섹션 상단 또는 중간에 배치되는 이미지 1장 | 필수 |
| **텍스트 블록** | 본문 내용 (리치텍스트 편집 가능, 단문 위주 자동 생성) | 필수 |
| **Q&A 블록** | GEO 대응용 질문-답변 구조 블록 (선택적 삽입) | 선택 |
| **핵심 요약 블록** | 글 마지막 섹션에 3줄 요약 자동 생성 (GEO 대응) | 마지막 섹션 |
| **섹션 컨트롤** | 추가(+) / 삭제(🗑) / 드래그 핸들(⠿) — 섹션 좌측 hover 시 노출 | 항상 표시 |

#### 3.2-E 섹션 관리 인터랙션

| 기능 | 동작 방식 | UX 디테일 |
|------|-----------|-----------|
| **섹션 추가** | 본문 하단 '+ 섹션 추가' 버튼 또는 섹션 간 + 버튼 클릭 | AI가 앞 섹션 흐름·C-Rank 주제 맥락 파악 후 다음 내용 자동 생성 |
| **섹션 삭제** | 섹션 좌측 🗑 버튼 클릭 → 토스트 확인 후 삭제 | 5초간 Undo 제공, 삭제 전 흐릿하게 미리보기 |
| **순서 변경** | 섹션 좌측 ⠿ 핸들을 드래그앤드롭으로 위아래 이동 | 드래그 중 대상 섹션 반투명 처리, 드롭존 파란 선 강조 |
| **섹션 복제** | 섹션 우측 … 메뉴 → '복제' 선택 | 이미지·텍스트 동일 복사 후 바로 아래 삽입 |
| **섹션 접기** | 섹션 우측 ^ 버튼으로 접기/펼치기 | 긴 문서 편집 시 스크롤 효율화 |

#### 3.2-F 섹션 이미지 등록 방식

| 방식 | 동작 | UX 디테일 |
|------|------|-----------|
| **드래그&드롭** | 로컬 이미지 파일을 이미지 블록 위로 드래그 | 드롭존 점선 강조 + 업로드 진행률 표시 |
| **복사&붙여넣기** | 클립보드 이미지를 이미지 블록 클릭 후 Ctrl+V (Mac: ⌘+V) | 스크린샷·웹 이미지 즉시 붙여넣기 지원 |
| **파일 탐색기** | 이미지 블록 클릭 시 파일 선택 다이얼로그 오픈 | jpg·png·webp·gif 지원, 최대 10MB |
| **AI 자동 생성** | 이미지 블록 내 '✨ AI 생성' 버튼 클릭 | Imagen 3 호출, 섹션 텍스트 맥락 기반 프롬프트 자동 구성 |

| **이미지 교체** | 등록된 이미지 hover → '교체' 버튼 노출 | 위 방식 중 선택하여 재등록 |

- **이미지 미등록 상태**: '📷 추천 사진: [섹션 맥락 기반 가이드 문구]' 플레이스홀더 자동 표시
- **이미지 위치 토글**: 섹션 내 이미지-텍스트 상하 순서를 버튼 1회 클릭으로 전환
- **이미지 리사이즈**: 네이버 블로그 최적 너비(966px) 기준 자동 처리, 비율 유지
- **텍스트 블록**: 문단별 '다시 쓰기' / '길게 늘리기' / '요약하기' AI 보조 버튼
- **SEO 점수 패널**: C-Rank·D.I.A+·GEO 항목별 충족 여부를 우측 패널에서 체크리스트로 실시간 표시
- **모바일·데스크탑 미리보기**: 우측 패널에서 실시간 렌더링
- **네이버 에디터 호환 HTML 복사 버튼**

### 3.3 [탭 3] 인스타 카드뉴스

블로그 본문을 인스타그램 최적화 카드뉴스 슬라이드로 자동 변환합니다.  
각 슬라이드는 이미지·텍스트·디자인 요소를 하나의 단위로 편집합니다.

#### 카드뉴스 슬라이드 구조

| 구성 요소 | 설명 | 비고 |
|-----------|------|------|
| **커버 슬라이드** | 제목 + 브랜드 로고 + 배경 이미지로 구성된 첫 장 | 자동 생성, 필수 |
| **본문 슬라이드** | 핵심 문구 1개 + 보조 설명 + 배경 이미지 또는 색상 | N장, 추가·삭제 가능 |
| **마무리 슬라이드** | CTA 문구 + 계정 태그 + 행동 유도 (댓글·팔로우 등) | 자동 생성, 선택 |
| **슬라이드 컨트롤** | 추가(+) / 삭제(🗑) / 드래그 핸들(⠿) — 좌측 패널에서 관리 | 항상 표시 |

#### 슬라이드 자동 생성 로직 (AI 4단계)

| 단계 | 처리 내용 | 결과물 |
|------|-----------|--------|
| **1. 핵심 문구 추출** | Gemini가 블로그 본문에서 임팩트 있는 문장 5–10개 선별 | 슬라이드별 헤드카피 |
| **2. 보조 설명 생성** | 헤드카피를 1–2줄로 보충 설명하는 서브카피 생성 | 슬라이드별 바디카피 |
| **3. 이미지 배정** | 각 슬라이드 텍스트 맥락 기반 Imagen 3 이미지 자동 생성 | 슬라이드별 배경·삽입 이미지 |
| **4. 디자인 적용** | 선택한 템플릿·색상·폰트 스타일 자동 적용 | 완성 슬라이드 미리보기 |

#### 슬라이드 편집 기능

| 기능 | 동작 방식 | UX 디테일 |
|------|-----------|-----------|
| **슬라이드 추가** | 좌측 패널 하단 '+ 슬라이드 추가' 클릭 또는 슬라이드 사이 + 버튼 | 빈 슬라이드 또는 AI 자동 생성 선택 |
| **슬라이드 삭제** | 좌측 패널 슬라이드 카드 우클릭 → 삭제, 또는 🗑 버튼 | 5초 Undo 제공 |
| **순서 변경** | 좌측 패널 슬라이드 카드를 드래그앤드롭으로 위아래 이동 | 드래그 중 카드 그림자 효과, 드롭존 강조 |
| **텍스트 편집** | 우측 캔버스에서 텍스트 클릭 후 직접 인라인 편집 | 폰트·크기·색상·정렬 툴바 노출 |
| **이미지 교체** | 캔버스 이미지 클릭 → 드래그&드롭·붙여넣기·파일·AI 생성 선택 | 블로그 탭과 동일한 4가지 방식 |
| **배경 스타일** | 단색·그라디언트·이미지 배경 전환 선택 가능 | 색상 팔레트 우측 패널 제공 |
| **AI 카피 재생성** | 헤드카피·바디카피 옆 🔄 버튼 클릭 | 같은 맥락으로 대안 문구 3개 제시 |

#### 디자인 시스템

| 항목 | 옵션 | 비고 |
|------|------|------|
| **템플릿** | 미니멀·감성·정보형·브랜드형 등 10종 제공 | 커버~마무리 슬라이드 일괄 적용 |

- **우측 패널**: 전체 슬라이드 미리보기 + 개별 슬라이드 선택 시 상세 편집

- **내보내기**: 슬라이드별 PNG 개별 저장 또는 전체 ZIP 일괄 다운로드, PDF 내보내기

### 3.4 [탭 4] 숏폼 영상 대본 (릴스 / 쇼츠)

블로그 본문을 60–90초 분량의 숏폼 영상 대본으로 변환합니다.  
각 씬(Scene)은 화면 설명과 나레이션 대본의 조합으로 구성됩니다.

#### 씬(Scene) 구조

| 구성 요소 | 설명 | 비고 |
|-----------|------|------|
| **후킹 씬** | 시청자를 첫 3초 안에 붙잡는 질문·충격·공감 문구 | 자동 생성, 필수 |
| **본문 씬** | 핵심 메시지를 5–8초 단위로 쪼갠 씬 블록들 | N개, 추가·삭제·순서 변경 가능 |

#### 대본 자동 생성 로직 (AI 5단계)

| 단계 | 처리 내용 | 결과물 |

| **2. 후킹 생성** | 첫 3초 이내에 시청자를 잡는 임팩트 문구 3개 후보 제시 | 후킹 씬 선택지 |
| **3. 나레이션 작성** | 각 씬의 핵심 메시지를 자연스러운 구어체 대본으로 변환 | 씬별 나레이션 텍스트 |
| **4. 화면 설명 생성** | 나레이션에 맞는 화면 연출 가이드 자동 생성 | 씬별 비주얼 디렉션 |
| **5. 타임코드 배정** | 씬별 권장 재생 시간 자동 배분 (총합 60–90초 기준) | 씬별 [00:00–00:05] 표시 |

| **후킹 변경** | 상단 후킹 씬 '후보 보기' 버튼으로 3개 대안 중 선택 | A/B/C 후보를 나란히 비교 |

#### 플랫폼별 설정

| 플랫폼 | 권장 길이 | 비율 | 특이사항 |
